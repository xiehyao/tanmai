# 推荐的编译方案

## 🎯 最佳实践：命令行编译

**不需要VNC！** 配置好命令行编译工具后，工作流程非常简单：

### 工作流程

1. **开发代码**（在任何编辑器，如VS Code）
   ```bash
   # 通过SSH编辑代码，或使用编辑器远程编辑
   ```

2. **编译项目**（一条命令）
   ```bash
   cd /var/www/html/moodle/tanmai/frontend
   npm run build
   ```

3. **部署**（自动或手动）
   ```bash
   # 编译后的文件在 dist/h5/ 目录
   # 可以自动部署或手动复制
   ```

### 与VNC+HBuilderX的对比

| 方案 | 是否需要VNC | 自动化 | 速度 | 推荐度 |
|------|------------|--------|------|--------|
| VNC+HBuilderX | ✅ 需要 | ❌ 手动 | 慢 | ⭐⭐ |
| 命令行编译 | ❌ 不需要 | ✅ 可以 | 快 | ⭐⭐⭐⭐⭐ |

- 开发调试时的辅助工具
- 或首次配置时的临时方案
