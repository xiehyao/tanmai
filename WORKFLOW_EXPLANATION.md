# 工作流程说明

## ❓ 关于VNC和编译工作流

### 问题：每次修改都要登录VNC吗？

**答案：不需要！** 有更实用的方案。

## 🎯 推荐的工作流程

### 方案1：命令行编译（推荐，不需要VNC）

配置uni-app的CLI工具链后，可以通过命令行编译：

```bash
# 修改代码后，直接运行编译命令
cd /var/www/html/moodle/tanmai/frontend
npm run build

# 编译后的文件自动生成，无需VNC
```

**优势**：
- ✅ 不需要图形界面
- ✅ 可以自动化
- ✅ 适合CI/CD
- ✅ 速度快

### 方案2：开发时用VNC，部署时用命令行

- **开发调试**：通过VNC使用HBuilderX（如果需要图形界面调试）
- **生产部署**：使用命令行编译（自动化，无需VNC）

### 方案3：保持VNC连接（如果必须用HBuilderX）

如果必须使用HBuilderX：
- 可以保持VNC连接开启
- 通过SSH上传/修改代码
- 在HBuilderX中刷新项目
- 重新编译

## 💡 我的建议

**最佳方案**：配置命令行编译，这样：
- 不需要VNC（除非调试）
- 修改代码后直接运行编译命令
- 可以写脚本自动化
- 更符合生产环境工作流
